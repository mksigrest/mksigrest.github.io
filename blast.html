Circle Blast Game

Role: Programmer

Languagues Used: C#

Programs Used: Visual Studio, Github

What is the purpose of this application: This application is meant to simulate how an ordering kiosk for a RIT dining resturant could possibly work. There is a login/signup ability, which will keep your information throughout login and logout. You can add items to your order, where the prices will add up in the checkout screen.

What was the design process: This was a group project, and it allowed us to get comfortable using github as a collaboration tool rather than an individual storage space. I was in charge of designing the add to order pages, along with the item classes that would be added to each order. I designed a main food class, and a child class for drinks.

What did I learn in this design process: I learned about merge conflicts in github, how to resolve them, and how to successfully parse coding work among a group of people so that the project is sucessfully finished. I continued gaining knowledge in C# programming, with my job allowing me to do more reaserch into C# classes and child classes, learning more about inheritance.
